微企聊第三方信息共享清单

当你使用微企聊（以下简称“本应用”）时，部分服务由第三方提供。我们会在获得你的授权同意后，或在其他符合法律法规的情况下，与第三方共享相关信息。你可以通过以下清单了解第三方如何处理你的个人信息。

为了确保微企聊的稳定运行或实现相关功能，我们可能会接入由第三方提供的软件开发包（SDK）。我们会对接入的第三方SDK进行严格的安全监测，以保护用户的数据安全。请注意，第三方SDK可能因版本升级或策略调整而导致其个人信息处理方式发生变化，具体情况请参考其官方说明。你可以通过本应用提供的目录或链接，查看第三方的数据使用和保护规则。

在使用过程中，我们会收集设备运行信息、MAC地址等设备信息，以保证消息功能和性能分析的实时性。如果用户取消授权，我们将不再收集相关设备信息，并无法继续提供相应服务。用户关闭权限的决定不会影响之前基于授权所进行的个人信息处理。我们建议用户定期查看第三方的隐私政策，以全面了解相关第三方的隐私实践，确保个人信息的安全。

小米推送 SDK

第三方主体： 北京小米移动软件有限公司 共享的信息： 设备标识(OAID、加密的Android ID)、推送消息内容、设备信息(设备厂商、型号等)、通知栏设置 可能调用的权限：访问网络状态、获取设备信息、写入设备存储 使用场景： 离线收到新消息时进行通知提示 使用目的： 提供消息推送服务及时告知用户新消息到达 共享方式： SDK 于本机采集 第三方个人信息处理规则：《小米推送隐私政策》

华为推送 SDK（华为HMS）

第三方主体：华为软件技术有限公司 共享的信息：应用基本信息、应用内设备标识符（Android ID、OAID、AAID和Push Token）、设备的硬件信息、系统基本信息和系统设置信息 使用场景：离线收到新消息时进行通知提示 使用目的： 1、提供消息推送服务；2、统计华为推送SDK接口调用成功率。 共享方式：SDK 于本机采集 第三方个人信息处理规则：《SDK隐私安全说明》

vivo 推送 SDK

第三方主体：维沃移动通信有限公司 共享的信息：应用基本信息、应用内设备标识符（Android ID、OAID、vpush的regId）、设备的硬件信息、系统基本信息 使用场景：离线收到新消息时进行通知提示 使用目的：提供消息推送服务及时告知用户新消息到达 共享方式：SDK 于本机采集 第三方个人信息处理规则： 《vivo推送隐私政策》，《vivo推送SDK隐私安全说明》

OPPO 推送 SDK

第三方主体：OPPO广东移动通信有限公司 共享的信息：设备相关信息（如IMEI或OAID，Serial Number，IMSI，User ID，Android ID，Google Advertising ID, 手机Region设置，设备型号，手机电量，手机操作系统版本及语言）、使用推送服务的应用信息（如APP包名及版本号，运行状态） 使用场景：离线收到新消息时进行通知提示 使用目的：提供消息推送服务及时告知用户新消息到达 共享方式： SDK 于本机采集 第三方个人信息处理规则：《推送个人信息保护政策》

魅族推送 SDK

第三方主体：珠海星纪魅族信息技术有限公司 共享的信息：设备相关信息，如手机品牌、手机型号、系统版本、系统语言，以及设备标识符（通过相关标识符生成应用间不相关的用户标识：push ID） 使用场景：离线收到新消息时进行通知提示 使用目的：提供消息推送服务及时告知用户新消息到达 共享方式： SDK 于本机采集 第三方个人信息处理规则： 《魅族推送隐私政策》

Bugly Android SDK

第三方主体：深圳市腾讯计算机系统有限公司 共享的信息：进程信息（进程名和PID）和其他设备基本信息（GUID、手机型号、系统版本、系统 API 等级等） 使用场景：用户在使用本应用过程中 使用目的：崩溃分析和运行日志收集 共享方式：SDK于本机采集 第三方个人信息处理规则：《Bugly SDK个人信息保护规则》

荣耀推送 SDK

第三方主体：深圳荣耀软件技术有限公司或关联公司 共享的信息：应用基本信息、应用内设备标识符（Android ID、OAID、AAID 和 Push Token）、设备的硬件信息、系统基本信息和系统设置信息 使用场景：离线收到新消息时进行通知提示 使用目的：提供消息通知功能 共享方式：SDK 于本机采集

第三方个人信息处理规则：《荣耀推送SDK隐私声明》 和《荣耀推送服务协议》

根据用户活跃区域而有所不同。您可参阅荣耀推送服务隐私政策（荣耀手机设置 > 关于 > 法律信息 > 基础服务声明 > 推送服务声明）

更多关于本应用获取和使用设备信息的说明请查看 《微企聊隐私保护指引》。
